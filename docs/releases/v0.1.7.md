# CueShot v0.1.7

Release date: 2026-06-22

## Highlights

- Reworks first-launch onboarding into a guided permission setup flow.
- Requires Screen Recording and Accessibility before onboarding can be completed, while keeping Automation clearly optional.
- Adds an Automation setup action that triggers a harmless System Events request before opening the macOS Automation pane, so CueShot appears under System Events instead of showing an empty pane.
- Adds a one-click hide control beside Arm in the floating Capture Control.
- Changes that hide action to tuck CueShot away to the menu bar by hiding normal app windows and the floating control while keeping the top menu bar item active.
- Adds a Settings toggle to hide CueShot's Dock icon when the menu bar item is active.
- Documents the first-launch permission behavior, Automation row creation, and menu-bar-only presentation.

## Permission Notes

CueShot still cannot silently pre-grant macOS Privacy permissions. macOS owns Screen Recording, Accessibility, and Automation approval.

This release improves the setup path:

- Screen Recording is required for visible pixel capture, screenshots, and OCR.
- Accessibility is required for the capture listener and exact element/window targeting.
- Automation/System Events is optional and only powers the advanced visible paste handoff into Codex.
- The Automation button now performs a small System Events query first, which prompts macOS to create the CueShot -> System Events permission row before opening System Settings.

## Menu Bar Notes

CueShot can now behave more like a quiet menu bar utility:

- The floating Capture Control has an eye-slash button next to Arm.
- Clicking it hides the floating control and app windows but leaves the menu bar item available.
- Settings includes `Hide Dock icon when menu bar item is active` for users who prefer a menu-bar-only presence.
